Шаблон проектирования "Команда" (Command pattern) - это поведенческий шаблон проектирования, который позволяет
инкапсулировать запрос в виде объекта, делая его возможным для передачи в качестве аргумента в методы, вызовов по
расписанию, хранения в истории и т.д.

В шаблоне проектирования Команда создается класс-команда, который инкапсулирует запрос в виде объекта. Затем создается
класс-отправитель, который содержит ссылку на объект-команду. Отправитель может вызвать метод execute() на объекте-команде, чтобы выполнить запрос. Также может быть создан класс-приемник, который содержит методы для выполнения самого запроса.

Основные составляющие шаблона проектирования Команда:

Команда (Command) - это интерфейс или абстрактный класс, который определяет метод execute() для выполнения запроса.
Конкретная команда (Concrete Command) - это класс, который реализует интерфейс Command и предоставляет реализацию для
выполнения определенного запроса.
Отправитель (Invoker) - это класс, который содержит ссылку на объект-команду и может вызывать метод execute() на
объекте-команде, чтобы выполнить запрос.
Приемник (Receiver) - это класс, который содержит методы для выполнения самого запроса.
